## 二、分阶段实施计划（按 价值/成本 排序）

### 🔴 Phase A：市场环境分桶（顶层定调）—— 最高价值
**目标**：报告开头给出明确的市场阶段判断，为全文定调，并据此动态收缩仓位建议。

**数据**（全部 Tushare，已可用）：
- `index_daily`：上证/创业板/深成指 近20日 → 趋势、累计回撤、是否破位
- 全市场日线 → 站上MA5/MA20占比（市场广度）
- 涨停/跌停家数（已有，板块感知精确版）
- 连板高度（识别连续涨停股最高几板）

**输出**：
```
【市场环境】阶段=反抽 | 置信度=0.70 | 风控级别=risk_on | 建议单票仓位上限=0.15
依据：上证站上5日线占比72%，涨停171家，成交额3万亿放量...
```

**落地**：新增 `_market_regime_analysis()` 函数 + 盘后 prompt 顶部增加"市场环境"区块。

---

### 🔴 Phase B：新闻↔行情强制交叉验证（prompt层，低成本高价值）
**目标**：杜绝"被新闻带偏"——新闻看多的板块必须用资金/趋势验证，矛盾时以行情为准。

**落地**：改盘后 system prompt + 用户 prompt，强制要求：
- 每个板块结论必须同时引用"新闻催化"+"资金/趋势数据"
- 出现矛盾（如新闻利好但3日资金流出）必须标注"⚠️新闻与资金背离，以资金为准→回避"

**成本**：仅改 prompt，0 代码。

---

### 🟡 Phase C：拥挤度/追高风险量化
**目标**：量化"追高风险"，自动给过热标的降级。

**指标设计**：
- 个股拥挤度 = 涨幅分位 × 换手率分位 × 量比（越高越危险）
- 板块拥挤度 = 板块内涨停家数占比 + 资金集中在前3只的比例
- 次日风险惩罚分 = f(拥挤度, 连板高度, 偏离均线幅度)

**落地**：新增 `_crowding_score()`，在板块全景和个股榜标注"🔥过热"标签；prompt 要求过热标的只能"观察"不能"买入"。

---

### 🟡 Phase D：个股量化指标增强
**目标**：明日关注个股附完整量化画像，复用选股线已算的因子。

**指标**（复用 `c_stock_selection` + 千股千评）：
- RSI_14、VWAP偏离、MA结构、RPS强度（选股线已算）
- 千股千评：综合得分、机构参与度、人气排名（akshare，服务器已通）
- 资金流 1日/3日

**落地**：盘后报告为"明日关注3只"补充量化指标行 + 双买点（保守MA/激进收盘）。

---

### 🟢 Phase E：完整板块趋势评分体系
**目标**：给每个板块算 0-100 趋势评分 + 阶段分类（低吸/主升/退潮/decay），形成"板块阶段识别表"。

**说明**：这是最完整但最重的一块，需要历史多日板块数据沉淀。建议最后做。
