## 4. 因子计算规范（映射真实接口，标注状态）

记主题成分股集合 S，|S|=N。

### 4.1 确定口径（照实现即生效）
- **多周期资金（亿）**：`money_flow_Nd = Σ_{s∈S} Σ_{近N日} 主力净流入(万元) / 1e4`。主力净流入 = `(buy_lg+buy_elg)-(sell_lg+sell_elg)`（`moneyflow`）。
- **多周期涨跌（等权）**：`ret_Nd(s)=∏(1+pct_chg/100)-1`；`pct_chg_Nd=mean_S(ret_Nd)×100`。【C4 等权 vs 市值加权待定】
- **前复权均线广度**（本期新增核心）：取 `adj_factor` 还原前复权收盘，`MA_x(s)=mean(qfq_close, x日)`；`breadth_ma_x = count_S(qfq_close≥MA_x)/N×100`，x∈{3,5,10,20,30,60,90,144}。
- **Top100/300 拥挤度**：全市场按【当日主力净流入】降序取前 100/300=R100/R300；`top100_ratio=|S∩R100|/N×100`。【C6 排名键可配置】

### 4.2 【需校准】复合口径（给默认实现）
- **heat_score**：`100×(0.40·z_money+0.25·z_ret+0.20·z_breadth+0.15·z_top)`，z=全市场分位归一；权重抽常量配置。【C1】
- **money_flow_3d_norm**：`money_flow_3d/√sample_count×k`，k 待原站反解；备选 `/板块流通市值`。【C2】
- **人气体系**（依赖 D1）：个股「日内综合权重 w」由人气排名经单调递减变换得到；主题级：`pop_weight=mean(w)`，`pop_concentration_hhi=Σ(w_i/Σw)²×100`，`pop_fairness=1-Gini(w)`。**排名→权重变换函数待校准**（已知 rank27→0.19、rank92→0.10），实现为 `w=clip(a·exp(-rank/k),0,1)` 抽配置。【C3】
- **phase**：主升初期=ret_7d 低 & ret_3d 放大 & breadth_ma20 上行 & 资金净流入；加速/decay/震荡按多周期斜率+广度+资金派生。
- **nextday_risk_penalty**：`α·max(pct_chg_1d-阈值,0)+β·(phase==decay)+γ·拥挤度过高`（已有，按 PRD 对齐系数）。

## 5. 人气抓取定时任务设计（D1）

- **数据源**：akshare `stock_hot_rank_em`（东财人气榜，列：当前排名/代码/股票名称/…），Top100。
- **调度**：新增 cron 两次——`09:45` 写 `am_rank`，`15:05` 写 `pm_rank`（工作日）。
- **落表** `popularity_rank`：`(trade_date, ts_code, am_rank, pm_rank, am_weight, pm_weight, intraday_weight, equiv_rank, created_at)`。
- **派生**：`收盘-早盘 = pm_rank - am_rank`（负=日内人气上行=更热）；权重由 §4.2 变换函数。
- **降级**：抓取失败 → 当日人气字段标「数据缺失」，下游 pop_* 置空不置零。
- **历史**：仅从启用日起前向有数据；早于启用日的 trade_date 人气列空缺。

- **路由**：`GET /sectorscope` + `GET /api/sectorscope?date=`。**大量复用 `industry_flow`/`sector_analyzer`**。

## 8. LLM 解读层规范（继承 PRD §7）
- 盘后批量为热度达阈值的主题生成 `reason/news_evidence/decision_note`，**落库缓存**；大盘解读每日 1 次。
- 输入只喂已算好的因子事实 + 检索新闻（财联社精筛 + 博查权威源），明确「数字是既定事实，只解释不改写」。
- 输出强制 JSON：`reason / news_evidence[] / decision_note{tier,score,enter_conditions[],falsify_conditions[],factor_explain[]}`。
- 防幻觉：每条结论绑验证点/证伪条件；新闻必给来源；低置信显式「数据不足，暂不结论」。

## 9. 工程与验收
- **配置化**：阈值/权重/排名键/人气变换系数 → `app/config_factors.py` 常量，便于对照原站校准。
- **缓存/调度**：宽表盘后批量；LLM 落库；前端读缓存。
- **降级**：数据源超时缺失 → 跳过该主题并降 `sample_reliability`，不填零。
- **验收**：① 任取主题展开全字段，每个数字能用 §4 公式手算对上；② 三栏信号按规则可复现；③ 前复权广度在除权日不跳变；④ LLM 字段为缓存且带验证点；⑤ 全程无 mock。
