## 一、环境要求

| 项 | 要求 |
|---|---|
| Python | **3.11**（服务器 `.venv`） |
| 系统 | Linux（生产 Ubuntu·中国时区 `Asia/Shanghai`） / macOS（开发） |
| 依赖管理 | `pyproject.toml`（虚拟环境 `.venv`） |
| 核心第三方库 | `tushare` `akshare` `pandas` `numpy` `fastapi` `uvicorn` `jinja2` `pydantic`/`pydantic-settings` `langgraph` `openai`(接DeepSeek) `httpx` `requests` `click` |
| 前端静态 | 自托管 `app/web/static/`(5个·**无 CDN**)：`echarts.min.js`(1MB) `html2canvas.min.js`(198KB) `kline.js` `chat.js` `research_widget.js`。⚠️**这些 .js 被 `.gitignore` 排除(不在git里)**·从 git 克隆后需另行获取(下载 ECharts/html2canvas 官方版·或从生产服务器 scp)，否则图表/K线/长图功能空白 |

```bash
# 从零安装
cd astock-agent
python3.11 -m venv .venv
.venv/bin/pip install -e .          # 按 pyproject.toml 安装
cp .env.example .env                # 若无则按下表手建
# 填好 .env 后：
.venv/bin/python -m app.run verify  # 验证所有数据接口真实可用
.venv/bin/python -m app.run web     # 起 Web (http://localhost:8000)
```

## 六、部署与运维

### 6.1 systemd(Web 常驻)
```ini
# /etc/systemd/system/astock-web.service
WorkingDirectory=/home/ubuntu/astock-agent
ExecStart=/home/ubuntu/astock-agent/.venv/bin/python -m app.run web --host 0.0.0.0 --port 8000
Restart=always
```
```bash
sudo systemctl restart astock-web    # ⚠️仅收盘后/周末(见纪律)·~4-6秒绑端口
systemctl is-active astock-web
```

### 6.2 部署流程
```bash
# 部署 = scp(排除 .env)·后端.py改动需重启·模板auto_reload免重启
scp app/xxx.py ubuntu@<host>:/home/ubuntu/astock-agent/app/xxx.py
# curl 验证(不打印密码):
U=$(grep -E '^WEB_USERNAME=' .env|cut -d= -f2-); U=${U:-admin}
PW=$(grep -E '^WEB_PASSWORD=' .env|cut -d= -f2-)
curl -s -u "$U:$PW" http://127.0.0.1:8000/api/...
```

### 6.3 crontab（工作日全量·见全景文档第四节时间轴）
盘前(08:35 pool-check→09:00 pre)→盘中(watch-scan每1分/market-scan每3分)→盘后(16:15 post-quick→18:00 wide→18:30 run→18:45 stock-pool→19:25 warmup)。09:05 `restore_production_feed.sh` 恢复幕数据生产feed。

## 七、运维纪律（务必遵守·血泪总结）

| 纪律 | 说明 |
|---|---|
| **交易时段不重启** | 9:15-11:30/13:00-15:00 禁 restart(会清空盯盘滚动历史)·**仅收盘后/周末**。盘中只 scp 不重启(次日09:05 cron加载) |
| **改因子必bump版本** | 改因子计算逻辑必 bump `_FACTOR_TABLE_VERSION`(v24)否则读到旧坏缓存(全NaN) |
| **结构化事实走结构化源** | 解禁/减持/预告的日期/数量只从 Tushare(`share_float`等)·**禁 LLM 从网页断言**(江波龙事故) |
| **诚实验证不擦粉** | 新因子/信号先回测·IC≈0就不加·不输出胜率排序·不做方向性买卖建议(只描述+风险) |
| **数据带来源** | AI 给的每个数字带来源+日期+可核查·禁臆想 |
| **HTTP非HTTPS** | 生产站HTTP·`navigator.clipboard`不可用·前端复制需 execCommand 兜底 |
| **主力净流入=估算** | 非龙虎榜真钱·唯机构席位净买是真钱·分清标注 |
| **交易日历/涨跌停** | 板块感知限幅(主板10%/科创创业20%/北交所30%/主板ST 7-6起10%·见`_board_limit_pct`) |

## 八、常见排障

| 现象 | 排查 |
|---|---|
| 盯盘无数据"待接入" | 幕数据L1断流→查 `FULLPUSH_*`·`restore_production_feed.sh`·或 Sina 兜底中 |
| 网页点开慢 | 检查对应 warmup 缓存是否生成(`data_cache/xxx/{date}`)·未暖机则首点冷算 |
| 因子表全 NaN | 版本号没 bump·删 `data_cache/factor_table/{date}_{ver}.parquet` 重算 |
| LLM 返回空(荐股) | 国产LLM对"列/挑股票"有荐股过滤→用科普口吻+极简措辞·写完必测返空 |
| 报告数字可疑 | 对照结构化源核查(share_float/fina_indicator)·警惕LLM从网页臆造 |
