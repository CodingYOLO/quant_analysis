# 完整选股报告 × 选股池 融合设计文档 v1（路线B）

> 生成时间：2026-06-16 23:19:40
> 目标：让「盘后完整选股报告」站在「选股池」肩膀上——统一为一套科学选股逻辑。
> 完整报告 = **选股池候选(最关注) + 个股避雷 + 多空辩论 + LLM 深度复盘**。
> 铁律：**充分使用当天真实数据，且必须等数据更新就绪后才选股**。

---

## 0. 为什么融合（解决"片面"）
选股池已纳入：板块热度、板块/多周期资金、人气、Top100拥挤度、次日风险、多路交叉验证、前向追踪——
正是老 c_stock_selection 缺的维度。融合后只保留这一套，老的片面五步评分退役。

---

## 1. 融合后流水线（LangGraph）
```
a_market_gate(大盘状态/仓位/can_open)
  → b_theme_analysis(LLM主题 + sector_stats 板块热度/资金/拥挤/次日风险)
  → C节点[改造]: 调用 stock_pool 引擎 → 最关注候选 → 转 Candidate
  → d_risk_debate(对最关注 top N 跑多空辩论)
  → e_report[增强]: 量化表 + 个股详情 + 避雷 + **LLM 深度复盘**
```
- C 节点不再用老五步评分；改为 `build_stock_pool(trade_date, market_label)` → 取 `is_focus` 候选 → 映射 Candidate。
- 选股池本身已落库(stock_pool表)+写前向追踪；完整报告复用其候选，数据**同源不打架**。

## 2. 数据新鲜度保障（你强调的核心）
C 节点选股前先 **校验当天数据就绪**，否则不出报告（避免用半截数据）：
1. `daily(today)` 非空 且 `moneyflow(today)` 非空（资金约17:15入库）。
2. `daily_basic(today)` 非空（换手/市值/人气代理）。
3. 宽表 `theme_heat_all_in_one(today)` 已计算（主题热度/板块全景，18:00 cron）。
   - 若缺：C 节点**现算宽表**(build_industry_wide+concept) 再选股，确保 theme_pick/板块热度可用。
- 任一缺失 → 报告标注"数据未就绪，跳过/降级"，不编造。
- cron 时序已保证：`18:00 wide → 18:30 run`；本设计再加 guard 兜底手动运行。

## 3. 选股池 → Candidate 映射
| Candidate.factors | 来源(选股池/signals) |
|---|---|
| close/pct_change/turnover_rate | close / pct_chg / turnover |
| market_cap(亿) | circ_mv_yi |
| fund_flow_3d(万元) | main_flow_3d × 1e4 |
| rps50 / change_pct_7d | rps50 / change_7d |
| pullback_score | signals.pullback_score |
| rsi_14 / vwap_deviation / avg_amplitude_5d | **对最关注(小集合)补算**(复用 factors)，否则 0 |
| TradePlan.buy_conservative/aggressive | buy_low / buy_high |
| stop_loss / tp1 / tp2 / position_pct | 同名 |
| execution_checklist | 复用 `_generate_execution_checklist` |
| Candidate.theme / filters_passed | theme / sources+strategy_label |
| Candidate.risk_flags | 选股池 risk_flags + 避雷命中 |

## 4. 个股避雷融入选择（不止展示）
对最关注候选跑：
- 结构化：`market_extras`(减持/立案/问询/退市/业绩预亏)
- 网络舆情：`news_guard` 博查最新负面（权威源）
- 命中 → **降级"仅观察"或剔除**(影响 is_focus/仓位)，并在报告标红。
（当前 e_report 只展示避雷，本设计让它**反作用于选择**。）

## 5. 多空辩论
对最关注 top N（≤max_candidates，默认10）跑 `d_risk_debate`（v4-flash，JSON 多空+软否决）。

## 6. LLM 深度复盘（报告增强）
报告末尾加 **LLM 综合点评**（接地式，复用 detail_common 博查+反编造）：
- 今日大盘环境(复用 market_env/regime) → 主线板块(宽表热度) → 为何选这些票(横向龙头对比) → 风险提示。
- 只解释已算因子+真实新闻，附来源，不预测涨跌、不输出胜率。

## 7. 与"选股池页(Tab2)"的分工
- **Tab2 选股池**：展示全池(候选+最关注)，多策略筛选、T+N 追踪，量化为主。
- **完整选股报告(推送)**：聚焦**最关注**，加 避雷否决 + 多空辩论 + LLM 深度复盘，是"可执行版"。
- 数据同源（同一 build_stock_pool 输出），口径一致。

## 8. 文件改动（预估）
- `c_stock_selection.py`：C 节点改造为调用选股池引擎 + Candidate 映射 + 数据新鲜度 guard + 避雷否决。（保留老函数备用/或退役）
- `stock_pool.py`：暴露给 pipeline 的接口（已就绪 build_stock_pool）；补"对最关注补算 rsi/vwap/amplitude"。
- `e_report.py`：增强——避雷影响排序展示 + 末尾 LLM 深度复盘段。
- 数据新鲜度 guard 工具（可放 run.py 或新 helper）。

## 9. 验收
- 完整报告候选 == 选股池最关注（同源）；每只票可溯源(策略/来源/置信度/避雷)。
- 数据未就绪时报告明确提示，不用旧/半截数据。
- LLM 复盘接地、附来源、无胜率排序。
- 推送(邮件/微信)正常。
