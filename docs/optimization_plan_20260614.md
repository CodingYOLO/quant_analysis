# A股Agent系统 — 全面优化方案
**制定时间：2026-06-14**
**对标参考：吴川AI量化系统**

---

## 核心差距总结

通过对比吴川系统输出，我们识别出 **4大类、15个具体改进点**。
按"价值高且实现快"优先排序。

---

## 第一阶段：因子升级（2~3小时，纯量化，无需依赖）

### 🔴 O1: 新增 RSI_14 因子
**位置**：`app/factors.py` + `app/nodes/c_stock_selection.py`
**为什么**：吴川 `RSI_14=63.18` 是标配指标，判断超买超卖。MACD 是形态信号，RSI 是动量强度，两者互补，不重复。
**实现**：`rsi()` 函数已存在于 `factors.py`，只需在选股流水线里计算并加入 `StockFactors`。
**输出**：`RSI_14=63` 显示在每只股票里，`RSI>70` 为过热警告，`RSI<30` 为超卖低吸。

### 🔴 O2: 新增 VWAP 偏离率
**位置**：`app/factors.py`（`vwap_position()` 已有）→ 接入 `StockFactors`
**为什么**：吴川 `VWAP位置=0.16` 表示股价在VWAP上方16%，量化追高风险。我们有VWAP但没算偏离率。
**实现**：将 `vwap_position()` 结果存入 `StockFactors.vwap_deviation`。
**影响**：偏离>15% → 追高风险标注；偏离<0% → 低于主力成本，低吸机会。

### 🔴 O3: 新增 7日涨跌幅 `change_pct_7d`
**位置**：`app/nodes/c_stock_selection.py`
**为什么**：吴川 `change_pct_7d=6.39`，判断短期是否涨幅过大。当日涨幅看不出趋势，7日能看出是否过热。
**实现**：从 `close_m` 取7日前后比较，一行代码。

### 🟡 O4: 市场状态置信度
**位置**：`app/nodes/a_market_gate.py` + `app/state.py`
**为什么**：吴川 `置信度=0.70`，让用户知道判断有多可靠。退潮反抽的置信度比主升/衰退低。
**实现**：`MarketRegime` 加 `confidence: float` 字段，根据多个指标共识度计算（站MA5占比接近边界值时置信度低）。

## 第二阶段：报告升级（3~4小时，高视觉价值）

### 🔴 O5: 隔夜风险总览表
**位置**：`app/nodes/e_report.py`
**为什么**：吴川报告里最实用的表格之一。一眼看清所有主题的次日风险。我们所有数据已有，只是没有汇总展示。
**格式**：
```
| 主题 | 类型 | 热度 | 1日涨跌 | 3日资金(亿) | 人气集中 | 次日风险 | 风控提示 |
|---|---|---|---|---|---|---|---|
| 小金属 | 行业 | 81 | +12.4% | +12.4 | 22% | 0 | ✅ 可关注 |
| 半导体 | 行业 | 85 | -0.8% | -96.4 | 5% | 45 | ⚠️ 资金流出 |
```
**数据来源**：`state.sector_stats`（已有全部字段），只需重新排版。

### 🔴 O6: 新闻-量化交叉验证（最关键改进）
**位置**：`app/nodes/b_theme_analysis.py` + `app/nodes/e_report.py`
**为什么**：吴川最精华的逻辑——新闻说"半导体热"，但量化说"退潮+资金流出"，两者结合才是真相。我们现在新闻和量化分开展示，没有交叉验证。
**实现**：在 `_build_theme_objects()` 里，已知 `sector_stats`，把量化验证结论写进 `Theme.evidence`：
```python
# 现在（仅新闻）
evidence = ["英伟达Vera CPU发布"]

# 升级后（新闻+量化验证）
evidence = [
  "⚡ 新闻：英伟达Vera CPU发布（事件催化）",
  "📊 量化验证：热度85，但3日资金流出96亿，趋势3日变化-13",
  "⚠️ 结论：新闻热+量化弱，退潮反抽，谨慎追高"
]
```
**输出**：报告里每个主题都有"新闻说什么 + 量化说什么 + 最终结论"三层。

### 🟡 O7: 个股"行情联动分析"区块
**位置**：`app/nodes/e_report.py`
**为什么**：吴川有"潜伏改善"和"风险主题"两个分类，我们只有候选股列表，缺乏这种定性分类。
**实现**：基于 `sector_stats` 里的 `phase + delta_3d` 自动分类。

## 第三阶段：概念板块（需VPN/服务器，影响最大）

### 🔴 O8: 接入概念板块（东方财富）
**位置**：`app/sector_analyzer.py` + `app/data/akshare_provider.py`
**为什么**：吴川分析"航天""低空经济""磷化工"等概念板块，我们只用 Tushare 的行业分类（半导体、铜等），**缺少市场真正热炒的概念维度**。概念板块和行业板块是完全不同的两个维度。
**数据**：akshare `stock_board_concept_name_em()` + `stock_board_concept_cons_em(symbol)`
**障碍**：本地VPN阻断，部署国内服务器后自动生效。先把代码写好，接口有了直接生效。
**影响**：这是最大的单一改进，能让主题分析从"行业维度"升级到"概念热点维度"。

### 🟡 O9: 概念热度趋势评分 `trend_score`
**位置**：`app/sector_analyzer.py`
**为什么**：吴川 `趋势评分=21.41` + `3日变化=-13.04`，是0-100的连续评分，比我们的阶段标签（升温/退潮）更精细。
**实现**：基于5日资金流、MA20广度、连板高度、涨跌幅，加权计算0-100的趋势评分（类似已有的 heat_score，但加入价格动量维度）。

## 第四阶段：个股深度分析（中等工作量）

### 🔴 O10: 千股千评人气排名（等服务器）
**位置**：`app/nodes/c_stock_selection.py`
**为什么**：吴川 `人气最新排名=167`（全市场前167名）和 `人气排名改善_3日=214`（3日上升214名），是**唯一正期望策略（事件动量）的核心信号**。排名快速提升 = 市场开始关注这只股票。
**状态**：代码已写，接口已测试，VPN限制。服务器部署后立即生效。

### 🟡 O11: 庄家控盘比例
**位置**：同上
**为什么**：吴川 `庄家=67.18`，越高越安全（主力不会轻易砸盘）。同样来自千股千评，等服务器。

### 🟡 O12: 实时新闻风控（per stock）
**位置**：`app/nodes/d_risk_debate.py`
**为什么**：吴川对每只候选股单独搜新闻，给出"风险分"和"政策催化分"。我们的 Bear Agent 只用量化数据，缺乏实时新闻验证。
**实现**：在 Bear Agent 里调用 `provider.get_stock_news(ts_code)`，让LLM评估是否有近期负面新闻。
**成本**：每只股约1次 DeepSeek-flash 调用，每日约 ¥0.005。

## 第五阶段：历史学习（长期积累）

### 🟡 O13: 主题历史胜率标签
**位置**：新建 `app/history_tracker.py`
**为什么**：吴川 `次日(T+1)胜率=25%` 标注在每个主题上，让用户知道这个主题历史表现。
**实现**：每日运行后，记录 (主题, 日期, 候选股) → 次日涨跌幅，逐步积累胜率数据。
**数据库**：SQLite，自动更新。

### 🟡 O14: 市场情绪综合分
**位置**：`app/nodes/a_market_gate.py`
**为什么**：吴川 `情绪分=21.70`，综合多个指标得出0-100的情绪分，比6态标签更细腻。
**实现**：加权组合（涨停家数权重30%+MA5占比权重30%+北向资金权重20%+连板高度权重20%）。

### 🟡 O15: 个股多日追踪（持仓跟踪）
**位置**：新建 `app/tracker.py`
**为什么**：吴川有"自选"模块，对选入的股票持续追踪。我们每天重新选股，没有连续性。
**实现**：记录入选日期，每日计算浮盈浮亏，到达止盈/止损价时在报告里标注。
